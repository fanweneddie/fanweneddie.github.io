<p style="text-align:left;"><strong>Building Global Index Table on Leveldb</strong>
	<span style="float:right;">Sept 2020  - present</span>
</p>

Advisor: Prof. [Yongkun Li](http://staff.ustc.edu.cn/~ykli/) ([ADSL](http://adsl.ustc.edu.cn/introduction/), USTC)

*When the memory is sufficient, leveldb does not fully utilize it for indexing. Therefore, we tried to read the index block into memory and build a global index table to accelerate indexing. I implemented the bloom filter in global index table and did some performance test. The ongoing project is [here](https://github.com/fanweneddie/global_index_table/tree/FanWen_BF).*

<p style="text-align:left;"><strong>Optimization on a Static Taint Analysis Tool for Java Application</strong>
	<span style="float:right;">July - Oct 2021</span>
</p>

Advisor: Prof. [Tianyin Xu](https://tianyin.github.io/) (UIUC)

*As a static taint analysis tool, cflow has some problems such as non-deterministic output and high false-positive rate. I checked the outputs for many times and found that they were caused by non-deterministic taint propagation and lack of use-checking. Later, I fixed the bug of non-deterministic output and reduced false-positive cases. The code is [here](https://github.com/fanweneddie/cflow/tree/dev).*
